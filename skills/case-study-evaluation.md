## Objective

Track case study submissions, assess them thoroughly, identify missing parts, and keep Ayesha proactively informed. Candidates have 48 hours to complete after passing values call.

**Pipeline Position:** Values Call → **Case Study Submission** → GWC Interview

---

## SOP Steps (8-Step Process)

### Step 1: Check Who Was Sent a Case Study

In Markaz:
- Open each candidate profile
- Scroll to the **Candidate Communication** section
- Confirm whether a case study was sent
- Note the send date (48-hour deadline starts here)

---

### Step 2: Check Whether the Candidate Submitted It

On the profile, verify whether:
- The candidate submitted text directly in the profile, OR
- Uploaded a document/file in the submission section

Note the submission timestamp.

---

### Step 3: Double-Check Gmail

Also check Ayesha's Gmail for emails with subject lines like:
- "New Case Study Submission"
- "[Candidate Name] — Case Study"
- Similar variations

**Critical rule:** You must check **BOTH Markaz AND Gmail**. Do not rely on only one source. Candidates may submit via email, via Markaz form, or both.

---

### Step 4: Download the Submission

Retrieve and download the submitted assignment.
- If PDF: extract text if needed
- If doc/spreadsheet: download and open
- If email attachment: extract and save
- If Markaz form text: copy to file for review

Have the submission ready for detailed review.

---

### Step 5: Read the Original Assignment First

Before evaluating the candidate's work:
- Read the original case study prompt that was sent to the candidate
- Understand:
  - The exact questions asked
  - Expected deliverables (document format, file count, sections required)
  - Scope of work required
  - Any specific constraints or rubrics mentioned

Do not evaluate the candidate's answer until you know what was asked.

---

### Step 6: Read the Candidate Submission Thoroughly

Review every submission manually and in full.
- Do not skip sections
- Read all pages/documents
- If multiple candidates submitted for one role, read each one one-by-one
- Note: Do not rush. Read carefully. Understand what the candidate actually wrote.

---

### Step 7: Evaluate Quality and Completeness

Assess:
- Quality of answers (depth, clarity, thought process)
- Completeness (all sections addressed)
- Whether candidate addressed all required parts
- Whether any piece is missing or incomplete
- Depth of thinking (not just surface answers)

Document findings clearly.

---

### Step 8: Flag AI Use or Weak Effort

If relevant, note concerns such as:
- Likely AI-generated answers (generic tone, lack of specificity, boilerplate structure)
- Shallow thinking (surface-level analysis, no evidence of real work)
- Low effort (incomplete answers, minimal engagement)
- Incomplete depth (answers that should be detailed but are brief)
- Weak motivation (phoned-in responses, no genuine interest)

These concerns can be flagged in the candidate's section in the report as a visible warning to Ayesha.

---

## Automation Requirement

As soon as a candidate submits a case study:

1. **Coco automatically reads it**
2. **Identifies whether any part is missing**
3. **Sends an email to Ayesha only if anything is incomplete**

**Email Logic (Template):**
```
Subject: [Candidate Name] Case Study Submitted — Missing Parts

Hi Ayesha,

[Candidate Name] (App ID: ####) has submitted their case study for [Role].

Missing or incomplete parts:
- [Part 1: specific gap description]
- [Part 2: specific gap description]
- [Part 3: specific gap description]

Please flag this to the candidate if they should resubmit a complete version.

—Coco
```

**When to send this email:**
- Candidate submitted but one or more required sections are blank/incomplete
- Candidate submitted but file format is wrong (e.g., sent .txt instead of .pdf)
- Candidate submitted but only answered 2 of 4 questions
- Any situation where the submission is not fully complete

**When NOT to send this email:**
- Submission is complete (even if quality is poor — that's for Step 7 assessment)
- Candidate submitted successfully and all parts are there

---

## Weekly Reporting Requirement

Coco should also proactively report every week (without Ayesha asking):

**Report includes:**
1. **Who has submitted their case study** (completed submissions this week)
   - Names, App IDs, roles
   - Quality flag if AI-use or weak effort detected (from Step 8)

2. **Who is overdue** (sent case study but 48+ hours passed, no submission)
   - Names, App IDs, roles
   - Days overdue
   - Recommendation: "Suggest follow-up email"

3. **Who needs follow-up** (various reasons)
   - Submitted but incomplete (flagged by automation)
   - Submitted but quality concerns (AI, weak effort)
   - Not yet sent case study but should have been (timing issues)

**Format:** Concise summary email to Ayesha, e.g.:

```
Subject: Case Study Status Report — Week of [Date]

Hi Ayesha,

**Submitted This Week (5):**
- Ahmed Khan (App ID 1234, Job 35) — submitted, complete, quality OK
- Bilal Malik (App ID 1235, Job 36) — submitted, complete, some AI indicators flagged
- [...]

**Overdue (3):**
- Fatima Ahmed (App ID 1250, Job 35) — 2 days overdue, sent on [date]
- Ghulam Hassan (App ID 1251, Job 36) — 1 day overdue, sent on [date]
- [...]

**Needs Follow-up (2):**
- Hulalah Khan (App ID 1252, Job 35) — submitted but missing Exercise 3
- Iqbal Ahmed (App ID 1253, Job 36) — submitted but likely AI-generated

Let me know if you'd like me to follow up with any candidates.

```

## Non-Negotiable Rules

1. **Check BOTH sources** — Markaz AND Gmail. Never rely on one only.

2. **Read the original assignment first** — Cannot evaluate an answer without knowing the question.

3. **Read submission fully** — Do not skip sections. Read every page.

4. **Read one-by-one** — If multiple candidates, review each independently. Don't batch or skim.

5. **Assess completeness explicitly** — Name which parts are missing, if any. Be specific.

6. **Flag AI and weak effort** — Honest assessment. Note if you detect AI-generated or low-effort work.

7. **Automation is mandatory** — Auto-flag incomplete submissions to Ayesha immediately. Don't wait.

8. **Weekly proactive reporting** — Don't wait for Ayesha to ask. Report who submitted, who's overdue, who needs follow-up.

9. **All flagged submissions are urgent** — If anything is missing, email Ayesha same day/next morning.

10. **Keep dates/times noted** — Track when case study was sent, when it was due, when it was submitted. Use these in reports.

---

## Common Mistakes

1. **Checking only Markaz** — Candidate submitted via email, not form. Missed it.

2. **Checking only Gmail** — Candidate submitted via Markaz, not email. Missed it.

3. **Reading original assignment after** — Evaluated answer without knowing question. Wrong assessment.

4. **Skipping parts of submission** — Read first 2 exercises, assumed rest was fine. Actually missing Exercise 4.

5. **Batching multiple candidates** — Read all 5 submissions in a row without stopping. Blended impressions, details missed.

6. **Missing the automation step** — Identified incomplete submission but didn't email Ayesha immediately. Day passed.

7. **No weekly reporting** — Waited for Ayesha to ask about status. Reactive instead of proactive.

8. **Being vague about gaps** — "Submission is incomplete" without saying WHAT is missing. Ayesha has to ask for details.

9. **Not flagging AI/weak effort** — Noticed suspicious patterns but didn't document them in assessment.

10. **Wrong file format not caught** — Candidate sent .txt instead of .pdf. Didn't flag as problem.
