# Skill: Report Generation — Talent Acquisition

## Purpose
Generate a structured, professional candidate screening report for hiring managers.

## Prerequisites
- Candidate scoring complete (see skills/cv-screening.md)
- All candidates ranked by composite score
- Top recommendation identified

## Output File
Save to: `output/YYYY-MM-DD-[position-name]-screening-report.md`
Example: `output/2026-02-26-senior-engineer-screening-report.md`

## Report Template


## Generating Charts (optional, for large pools)
If the CV pool is > 50 candidates, add a score distribution chart:
```python
import matplotlib.pyplot as plt
import pandas as pd

df = pd.DataFrame(candidates)
fig, ax = plt.subplots(figsize=(10, 5))
ax.hist(df['composite_score'], bins=20, color='steelblue', edgecolor='white')
ax.set_title('Candidate Score Distribution — [Position Name]')
ax.set_xlabel('Composite Score (%)')
ax.set_ylabel('Number of Candidates')
ax.axvline(x=60, color='red', linestyle='--', label='Shortlist threshold (60%)')
ax.legend()
plt.tight_layout()
plt.savefig(f'output/{position_name}-score-distribution.png', dpi=150)
plt.close()
```

## Common Mistakes
- **Vague Top Recommendation**: "Good candidate" is useless. Quote specific evidence.
- **No gaps section**: Every candidate has gaps. Be honest — this helps the manager.
- **Forgetting over-budget flags**: Strong-but-over-budget candidates must be visible.
- **Not stating methodology**: Hiring managers need to know how scores were calculated.
- **Wrong filename**: Always include position name + date for easy archive search.
